## 1. 排序是什么

排序就是把一组数据按照某种规则重新排列。

例如从小到大：

```text
原数据：5 2 8 1 3
排序后：1 2 3 5 8
```

如果题目只说：

> 将 n 个元素从小到大排序。

严格来说，时间复杂度不能唯一确定，因为不同排序算法复杂度不同。

## 2. 内部排序和外部排序

| 类型 | 含义 |
|---|---|
| 内部排序 | 数据都能放进内存，在内存中完成排序 |
| 外部排序 | 数据太大，内存放不下，需要借助磁盘等外存 |

408 基础题里常考内部排序。

## 3. 稳定性

稳定性看的是：

> 两个元素关键字相等时，排序后它们原来的先后顺序是否保持不变。

例如：

```text
原数据：3a 1 3b 2
```

如果排序后是：

```text
1 2 3a 3b
```

`3a` 仍然在 `3b` 前面，这是稳定的。

如果排序后是：

```text
1 2 3b 3a
```

相同关键字元素的相对顺序变了，这是不稳定的。

## 4. 原地排序

| 概念 | 含义 |
|---|---|
| 原地排序 | 只需要常数级额外空间，空间复杂度通常是 `O(1)` |
| 非原地排序 | 需要额外数组或较多辅助空间 |

例如选择排序、堆排序通常是原地排序；归并排序通常要额外数组，所以不是典型原地排序。

## 5. 十大排序速查表

| 排序算法 | 最好 | 平均 | 最坏 | 空间 | 稳定性 | 核心理解 |
|---|---:|---:|---:|---:|---|---|
| 冒泡排序 | `O(n)` | `O(n^2)` | `O(n^2)` | `O(1)` | 稳定 | 相邻比较，大的往后冒 |
| 选择排序 | `O(n^2)` | `O(n^2)` | `O(n^2)` | `O(1)` | 不稳定 | 每趟选最小值放到前面 |
| 插入排序 | `O(n)` | `O(n^2)` | `O(n^2)` | `O(1)` | 稳定 | 像整理扑克牌，插入有序区 |
| 希尔排序 | 和增量有关 | 和增量有关 | 常见可到 `O(n^2)` | `O(1)` | 不稳定 | 分组插入排序，逐步缩小间隔 |
| 归并排序 | `O(n log n)` | `O(n log n)` | `O(n log n)` | `O(n)` | 稳定 | 分治：拆开，再合并 |
| 快速排序 | `O(n log n)` | `O(n log n)` | `O(n^2)` | 平均 `O(log n)` | 不稳定 | 选基准，左右分区 |
| 堆排序 | `O(n log n)` | `O(n log n)` | `O(n log n)` | `O(1)` | 不稳定 | 建堆，反复取堆顶 |
| 计数排序 | `O(n + k)` | `O(n + k)` | `O(n + k)` | `O(n + k)` | 通常稳定 | 统计每个值出现次数 |
| 桶排序 | 接近 `O(n + k)` | 依赖分布 | 可能退化 | `O(n + k)` | 看桶内排序 | 分桶，桶内排序，再合并 |
| 基数排序 | `O(d(n + r))` | `O(d(n + r))` | `O(d(n + r))` | `O(n + r)` | 稳定 | 按个位、十位、百位多轮排序 |

符号说明：

- `n`：元素个数。
- `k`：数据范围或桶数量。
- `d`：最大数字的位数。
- `r`：基数，例如十进制 `r = 10`。

## 6. 冒泡排序

核心思想：

> 相邻两个元素比较，顺序不对就交换；每一趟把当前最大值放到后面。

例子：

```text
原数组：5 3 8 4 6
第1趟：3 5 4 6 8
第2趟：3 4 5 6 8
```

为什么是 `O(n^2)`：

```text
第 1 趟最多比较 n - 1 次
第 2 趟最多比较 n - 2 次
...
第 n - 1 趟最多比较 1 次
```

总次数：

```text
(n - 1) + (n - 2) + ... + 1 = n(n - 1) / 2
```

去掉常数和低阶项：

```text
O(n^2)
```

如果加了“本趟没有交换就提前结束”的标记，最好情况有序数组只扫一遍，是 `O(n)`。

稳定性：

> 稳定。相等元素不交换，相对顺序不变。

C++ 骨架：

```cpp
void bubbleSort(vector<int>& a) {
    int n = a.size();
    for (int i = 0; i < n - 1; i++) {
        bool changed = false;
        for (int j = 0; j < n - 1 - i; j++) {
            if (a[j] > a[j + 1]) {
                swap(a[j], a[j + 1]);
                changed = true;
            }
        }
        if (!changed) break;
    }
}
```

考试抓手：

```text
相邻比较
稳定
最好 O(n)，平均/最坏 O(n^2)
```

## 7. 选择排序

核心思想：

> 每一趟从未排序部分中选出最小值，放到未排序部分的开头。

例子：

```text
原数组：64 25 12 22 11
第1趟：11 25 12 22 64
第2趟：11 12 25 22 64
第3趟：11 12 22 25 64
```

为什么始终是 `O(n^2)`：

选择排序不管原数组是否有序，都要在未排序区里找最小值：

```text
(n - 1) + (n - 2) + ... + 1 = O(n^2)
```

稳定性：

> 不稳定。因为最小值会和前面的元素直接交换，可能把相同关键字的顺序打乱。

例如：

```text
原数组：2a 2b 1
选出 1 后和 2a 交换：1 2b 2a
```

`2a` 和 `2b` 的相对顺序变了。

C++ 骨架：

```cpp
void selectionSort(vector<int>& a) {
    int n = a.size();
    for (int i = 0; i < n - 1; i++) {
        int minIndex = i;
        for (int j = i + 1; j < n; j++) {
            if (a[j] < a[minIndex]) minIndex = j;
        }
        swap(a[i], a[minIndex]);
    }
}
```

考试抓手：

```text
每趟选最小
比较次数固定
不稳定
最好/平均/最坏都是 O(n^2)
```

## 8. 插入排序

核心思想：

> 前面维护一个有序区，每次拿一个新元素，插到有序区的合适位置。

它很像整理扑克牌。

例子：

```text
原数组：5 2 4 6 1 3
插入 2：2 5 4 6 1 3
插入 4：2 4 5 6 1 3
插入 6：2 4 5 6 1 3
插入 1：1 2 4 5 6 3
插入 3：1 2 3 4 5 6
```

复杂度：

- 最好：原本有序，每个元素只比较一次，`O(n)`。
- 最坏：原本逆序，每次都要往前移动很多元素，`O(n^2)`。
- 平均：`O(n^2)`。

稳定性：

> 稳定。比较时只把比当前元素大的元素后移，相等元素不跨过去。

C++ 骨架：

```cpp
void insertionSort(vector<int>& a) {
    int n = a.size();
    for (int i = 1; i < n; i++) {
        int key = a[i];
        int j = i - 1;
        while (j >= 0 && a[j] > key) {
            a[j + 1] = a[j];
            j--;
        }
        a[j + 1] = key;
    }
}
```

考试抓手：

```text
基本有序时很快
稳定
最好 O(n)，平均/最坏 O(n^2)
```

## 9. 希尔排序

核心思想：

> 希尔排序是插入排序的改进版，也叫缩小增量排序。

它不是一步一步挪，而是先按较大的间隔 `gap` 分组，让元素能“大步移动”；等数组基本有序后，再用 `gap = 1` 做最后一轮插入排序。

例子：

```text
原数组：8 3 1 2 7 5 6 4
gap = 4：比较并整理相隔 4 的元素
gap = 2：继续整理相隔 2 的元素
gap = 1：普通插入排序，完成排序
```

复杂度：

> 希尔排序复杂度和增量序列有关，不像冒泡/选择那样固定。

常见结论：

```text
空间复杂度：O(1)
稳定性：不稳定
最坏情况：常见教材里通常记 O(n^2)
```

为什么不稳定：

分组排序时，相同关键字的元素可能被分到不同组里，经过远距离移动后，相对顺序可能改变。

C++ 骨架：

```cpp
void shellSort(vector<int>& a) {
    int n = a.size();
    for (int gap = n / 2; gap > 0; gap /= 2) {
        for (int i = gap; i < n; i++) {
            int key = a[i];
            int j = i;
            while (j >= gap && a[j - gap] > key) {
                a[j] = a[j - gap];
                j -= gap;
            }
            a[j] = key;
        }
    }
}
```

考试抓手：

```text
插入排序改进
缩小增量
不稳定
复杂度和 gap 序列有关
```

## 10. 归并排序

核心思想：

> 分治：先把数组一分为二，递归排好左右两半，再把两个有序数组合并。

例子：

```text
原数组：38 27 43 3 9 82 10
拆分：38 27 43 3 | 9 82 10
继续拆到单个元素
合并：27 38 | 3 43 | 9 | 10 82
继续合并：3 27 38 43 | 9 10 82
最终：3 9 10 27 38 43 82
```

为什么是 `O(n log n)`：

- 每次对半拆，层数是 `log n`。
- 每一层合并时，所有元素总共都会被处理一遍，是 `O(n)`。

所以：

```text
总时间 = 层数 × 每层工作量 = O(log n) × O(n) = O(n log n)
```

空间复杂度：

```text
需要临时数组辅助合并，通常是 O(n)
```

稳定性：

> 稳定。合并两个有序数组时，相等元素优先取左边数组的元素即可保持原顺序。

C++ 骨架：

```cpp
void merge(vector<int>& a, int l, int m, int r, vector<int>& temp) {
    int i = l, j = m + 1, k = l;
    while (i <= m && j <= r) {
        if (a[i] <= a[j]) temp[k++] = a[i++];
        else temp[k++] = a[j++];
    }
    while (i <= m) temp[k++] = a[i++];
    while (j <= r) temp[k++] = a[j++];
    for (int p = l; p <= r; p++) a[p] = temp[p];
}

void mergeSort(vector<int>& a, int l, int r, vector<int>& temp) {
    if (l >= r) return;
    int m = l + (r - l) / 2;
    mergeSort(a, l, m, temp);
    mergeSort(a, m + 1, r, temp);
    merge(a, l, m, r, temp);
}
```

考试抓手：

```text
分治
稳定
时间稳定 O(n log n)
空间 O(n)
适合外部排序
```

## 11. 快速排序

核心思想：

> 选一个基准 pivot，把比它小的放左边，比它大的放右边，然后递归排序左右两边。

例子：

```text
原数组：3 6 8 10 1 2
选 3 为基准
分区后：1 2 | 3 | 6 8 10
继续排左右部分
```

为什么平均是 `O(n log n)`：

- 如果每次基准比较居中，递归层数约 `log n`。
- 每一层分区都要扫描当前区间，总工作量约 `O(n)`。

所以平均：

```text
O(n log n)
```

为什么最坏是 `O(n^2)`：

如果每次基准都选到最小或最大，例如数组本来有序还总选第一个元素：

```text
每次只能分出 0 个元素和 n - 1 个元素
```

递归会退化成：

```text
n + (n - 1) + (n - 2) + ... + 1 = O(n^2)
```

稳定性：

> 不稳定。分区交换时，相同关键字元素可能改变相对顺序。

C++ 骨架：

```cpp
int partition(vector<int>& a, int l, int r) {
    int pivot = a[r];
    int i = l;
    for (int j = l; j < r; j++) {
        if (a[j] <= pivot) {
            swap(a[i], a[j]);
            i++;
        }
    }
    swap(a[i], a[r]);
    return i;
}

void quickSort(vector<int>& a, int l, int r) {
    if (l >= r) return;
    int p = partition(a, l, r);
    quickSort(a, l, p - 1);
    quickSort(a, p + 1, r);
}
```

考试抓手：

```text
平均快
不稳定
平均 O(n log n)，最坏 O(n^2)
递归栈平均 O(log n)，最坏 O(n)
```

## 12. 堆排序

核心思想：

> 利用堆这种近似完全二叉树结构。升序排序一般建大顶堆，每次把堆顶最大值放到数组末尾。

大顶堆特点：

```text
父结点 >= 左右孩子
```

数组下标关系：

```text
左孩子：2 * i + 1
右孩子：2 * i + 2
父结点：(i - 1) / 2
```

过程：

```text
1. 建大顶堆
2. 交换堆顶和最后一个元素
3. 堆大小减 1
4. 调整堆
5. 重复直到有序
```

为什么是 `O(n log n)`：

- 建堆可以做到 `O(n)`。
- 之后取出 `n - 1` 次堆顶，每次调整堆是 `O(log n)`。

总复杂度：

```text
O(n) + O(n log n) = O(n log n)
```

稳定性：

> 不稳定。堆顶和末尾元素反复交换，可能打乱相同关键字元素的相对顺序。

C++ 骨架：

```cpp
void heapify(vector<int>& a, int n, int i) {
    int largest = i;
    int left = 2 * i + 1;
    int right = 2 * i + 2;

    if (left < n && a[left] > a[largest]) largest = left;
    if (right < n && a[right] > a[largest]) largest = right;

    if (largest != i) {
        swap(a[i], a[largest]);
        heapify(a, n, largest);
    }
}

void heapSort(vector<int>& a) {
    int n = a.size();
    for (int i = n / 2 - 1; i >= 0; i--) heapify(a, n, i);
    for (int i = n - 1; i > 0; i--) {
        swap(a[0], a[i]);
        heapify(a, i, 0);
    }
}
```

考试抓手：

```text
选择排序思想的改进
不稳定
时间 O(n log n)
空间 O(1)
建堆 O(n)
```

## 13. 计数排序

核心思想：

> 不比较元素大小，而是统计每个值出现了多少次，再按顺序输出。

例子：

```text
原数组：4 2 2 8 3 3 1
统计：
1 出现 1 次
2 出现 2 次
3 出现 2 次
4 出现 1 次
8 出现 1 次
输出：1 2 2 3 3 4 8
```

适用条件：

```text
适合整数
数据范围不能太大
```

为什么是 `O(n + k)`：

- 扫原数组统计次数：`O(n)`。
- 扫计数数组：`O(k)`。
- 输出结果：`O(n)`。

所以：

```text
O(n + k)
```

稳定性：

> 可以稳定。使用前缀和并从后往前放入输出数组时，能保持相同元素原来的顺序。

C++ 简化版：

```cpp
vector<int> countingSort(const vector<int>& a) {
    if (a.empty()) return {};

    int minValue = *min_element(a.begin(), a.end());
    int maxValue = *max_element(a.begin(), a.end());
    vector<int> count(maxValue - minValue + 1, 0);

    for (int x : a) count[x - minValue]++;

    vector<int> result;
    for (int i = 0; i < (int)count.size(); i++) {
        while (count[i]--) result.push_back(i + minValue);
    }
    return result;
}
```

考试抓手：

```text
非比较排序
适合范围小的整数
时间 O(n + k)
空间 O(n + k) 或 O(k)
通常可稳定
```

## 14. 桶排序

核心思想：

> 按数据范围把元素分到多个桶里；每个桶内部排序；最后按桶顺序合并。

例子：

```text
原数组：0.42 0.32 0.33 0.52 0.37 0.47 0.51
桶3：0.32 0.33 0.37
桶4：0.42 0.47
桶5：0.52 0.51
桶内排序后合并：
0.32 0.33 0.37 0.42 0.47 0.51 0.52
```

什么时候快：

```text
数据分布比较均匀，每个桶里的元素少
```

什么时候慢：

```text
大量元素都落到同一个桶里，退化成桶内排序
```

复杂度：

桶排序没有一个绝对固定的复杂度，要看桶的数量、映射函数、桶内排序算法和数据分布。

常见记法：

```text
平均接近 O(n + k)
最坏可能退化到 O(n^2)
空间 O(n + k)
```

稳定性：

> 取决于桶内排序是否稳定，以及入桶/出桶是否保持顺序。

C++ 思路骨架：

```cpp
void bucketSort(vector<double>& a) {
    int n = a.size();
    vector<vector<double>> buckets(n);

    for (double x : a) {
        int index = min(n - 1, (int)(x * n));
        buckets[index].push_back(x);
    }

    for (auto& bucket : buckets) {
        sort(bucket.begin(), bucket.end());
    }

    int p = 0;
    for (auto& bucket : buckets) {
        for (double x : bucket) a[p++] = x;
    }
}
```

考试抓手：

```text
非比较排序思想
依赖数据分布
空间换时间
桶内排序影响整体性能和稳定性
```

## 15. 基数排序

核心思想：

> 按“位”排序。十进制整数可以先按个位排，再按十位排，再按百位排。

常见 LSD 基数排序：

```text
从低位到高位排序
个位 -> 十位 -> 百位 -> ...
```

例子：

```text
原数组：170 45 75 90 802 24 2 66
按个位排：170 90 802 2 24 45 75 66
按十位排：802 2 24 45 66 170 75 90
按百位排：2 24 45 66 75 90 170 802
```

为什么要求每一轮排序稳定：

按低位排完后，再按高位排。如果高位相同，必须保留低位已经排好的顺序。否则前面轮次的成果会被打乱。

复杂度：

如果最大数字有 `d` 位，每一位用计数排序，基数为 `r`：

```text
每一轮：O(n + r)
一共 d 轮：O(d(n + r))
```

稳定性：

> 稳定，前提是每一位使用稳定排序。

C++ 骨架：

```cpp
void countingSortByDigit(vector<int>& a, int exp) {
    int n = a.size();
    vector<int> output(n);
    vector<int> count(10, 0);

    for (int x : a) count[(x / exp) % 10]++;
    for (int i = 1; i < 10; i++) count[i] += count[i - 1];

    for (int i = n - 1; i >= 0; i--) {
        int digit = (a[i] / exp) % 10;
        output[count[digit] - 1] = a[i];
        count[digit]--;
    }

    a = output;
}

void radixSort(vector<int>& a) {
    if (a.empty()) return;
    int maxValue = *max_element(a.begin(), a.end());
    for (int exp = 1; maxValue / exp > 0; exp *= 10) {
        countingSortByDigit(a, exp);
    }
}
```

考试抓手：

```text
非比较排序
按位排序
每一轮必须稳定
时间 O(d(n + r))
适合整数或定长字符串
```

## 18. 例题：将 n 个元素从小到大排序

如果题目只写：

> 将 n 个元素从小到大排序。

严格来说，要看具体算法。

如果按冒泡排序推导：

```text
第 1 趟：比较 n - 1 次
第 2 趟：比较 n - 2 次
第 3 趟：比较 n - 3 次
...
第 n - 1 趟：比较 1 次
```

总比较次数：

```text
(n - 1) + (n - 2) + ... + 1
```

等差数列求和：

```text
n(n - 1) / 2
```

展开：

```text
(n^2 - n) / 2
```

时间复杂度去掉常数和低阶项：

```text
O(n^2)
```

所以：

> 如果题目按基础简单排序理解，通常把“排序 n 个元素”看作 `O(n^2)`。

如果题目明确说快速排序、归并排序、堆排序，通常按 `O(n log n)` 分析；但快速排序最坏是 `O(n^2)`。

## 19. 最容易考混的点

选择排序为什么不稳定：

```text
因为它会把最小值直接交换到前面，可能跨过相同关键字元素。
```

插入排序为什么稳定：

```text
相等时不往前插，只移动比它大的元素。
```

归并排序为什么需要 `O(n)` 空间：

```text
合并两个有序子数组时，需要临时数组保存合并结果。
```

快速排序为什么最坏 `O(n^2)`：

```text
每次基准都选得很差，分区极不平衡。
```

堆排序为什么不稳定：

```text
堆顶和末尾反复交换，相同关键字的相对顺序可能被打乱。
```

基数排序为什么要求稳定：

```text
低位排序结果要被高位排序保留下来。
```

## 20. 一句话总结

```text
冒泡：相邻交换，稳定。
选择：每趟选最小，不稳定。
插入：插入有序区，稳定，基本有序时快。
希尔：分组插入，不稳定。
归并：分治合并，稳定，空间 O(n)。
快速：基准分区，平均快，不稳定，最坏 O(n^2)。
堆排：建堆取顶，不稳定，空间 O(1)。
计数：统计次数，适合小范围整数。
桶排：按范围分桶，依赖数据分布。
基数：按位排序，要求每轮稳定。
```
