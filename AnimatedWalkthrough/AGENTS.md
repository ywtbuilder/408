# 408 算法动画演示 (AnimatedWalkthrough) 代理开发指南

为了确保所有数据结构算法动画演示页面均具备统一的视觉风格、交互逻辑与考研答题规范，在修改或新建演示项目时，请严格遵守以下开发指南：

## 1. 界面设计风格与配色方案 (Design & Aesthetics)
- **清爽简约风格**：页面整体以简洁、现代感为主，严禁使用杂乱、冗余的辅助文字与多余按钮，确保重点（代码追踪与图形变化）突出。
- **配色规范 (Harmonious Palette)**：
  - **背景色**：主背景使用浅色护眼配色，如 `Slate 50` (`#f8fafc`)。Canvas 画布区域使用柔和的网格点（Dotted Grid）作为底纹，增加空间感与对齐感。
  - **组件背景**：白色卡片配以轻微的阴影（`rgba(15, 23, 42, 0.04)`）和极细的浅灰色边框，呈现精致的扁平化卡片效果。
  - **数据结点**：
    - 普通元素结点：白色到浅蓝灰色渐变，灰色细边框。
    - 头结点 (Dummy Head)：淡紫色到紫粉色渐变，紫色边框，紫色文字。
    - 新分配结点：淡绿色渐变，翠绿色边框。
  - **指针徽章与高亮色**：
    - 尾指针 `p` (Rear)：玫瑰红 (`#f43f5e`) 徽章与对应虚线。
    - 新结点指针 `q`：翡翠绿 (`#10b981`) 徽章。
    - 临时指针 `r` / `t`：琥珀黄 (`#f59e0b`) 或蓝色 (`#3b82f6`)。
    - 激活路径：紫色 (`#8b5cf6`) 动态虚线箭头。

## 2. 交互逻辑与功能模块 (Interactive Features)
- **模式切换 (Mode Toggle)**：必须支持 408 考点中常见的对比模式（如“带头结点”与“不带头结点”），通过滑动或圆形胶囊按钮无缝切换。
- **数据操作**：
  - 支持数据输入（防越界/防呆输入设计），包含“入队/插入”、“出队/删除”等数据操作按钮。
  - 支持一键“清空”和“重置”当前数据结构。
- **播放控制器 (Playback Controls)**：
  - 核心包含：重置 (↺)、上一步 (◀)、播放/暂停 (▶/⏸)、下一步 (▶) 按钮。
  - 提供滑动条调节自动播放的速度（500ms ~ 3000ms 范围）。
- **实时步骤日志**：在画布头部显著位置以高对比度、清晰字体展示当前运行的具体步骤和对应的中文逻辑说明。

## 3. C++ 算法代码追踪 (Code Tracer)
- **408 答题标准**：右侧展示的代码段必须符合 408 考研大题的标准四段式规范（题目、算法思想、数据结构定义、算法实现、复杂度分析）。
- **同步追踪**：图形动画运行的每一步，右侧 C++ 代码的对应行必须以醒目的背景色（如琥珀色背景、左侧粗边框指示条）精准同步，并且在长代码时支持平滑滚动至可视区域。
- **语法高亮**：使用符合整体浅色风格的高亮方案，清晰区分 C++ 关键字、自定义类型、注释及字面量。

## 4. 技术栈限制 (Tech Stack Limits)
- 仅使用原生 Vanilla HTML、CSS、Javascript 构建。避免引入复杂的外部框架（如 TailwindCSS 等，除非明确指定），保证单页面演示在各种环境下的无缝运行。
